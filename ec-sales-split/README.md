# EC販売データ振り分け — 自社／メーカー変換（同時生成）

`item-data`（自社商品集計）と `ec-maker-only`（メーカー商品変換）を1本にまとめた派生アプリ。`index.html` 一枚のシングルページアプリ。

同じ**受注・売上データCSV**を1回アップロードするだけで、行ごとに**自社商品**か**メーカー（仕入れ）商品**かを自動判定し、それぞれ専用のルール・形式で変換した**2つのファイルを同時に生成**する。

公開URL: `https://elcommun.github.io/Claude-App/ec-sales-split/`

## 入力データ

| ファイル | 必須 | 役割 |
|---------|------|------|
| **受注・売上データ CSV** | 必須 | `販売店舗 / 出荷日 / 商品名 / 商品番号 / SKU管理番号 / 個数 / 商品単価 / 税込別 …` を含む全店舗の受注/売上一覧。Shift-JIS（CP932）想定。GoQ Systemからのダウンロード方法をアプリ内に記載。 |
| **商品データ xlsx** | 必須 | 自社品番マスタ・商品名・商品分類（「商品データ」「plaisir別注」「商品分類」シート）。`item-data` と同じ形式。自社／メーカー判定の自社品番マスタとしても使う。 |

## 振り分けルール（`item-data` 準拠）

各行の `商品番号` を内部品番に解決し（`extractProductCode` / `normalizeCode` ＋ Amazon/Qoo10 のハードコードマップ、`item-data`・`ec-maker-only` と共通のロジック）、

- 自社品番マスタ（商品データxlsxの品番一覧、および `ec-inventory/genka_master.js` の内蔵スナップショット）に一致
- `elco-` / `cal-` プレフィックス
- 自社確定接頭辞（`X?DR-` `CAL-` `EGP-` `LBG-` `AMP-C` `SFD-` `MK-` `CRB-` `WCL-`、大文字 `AK-`）
- 商品名に「エルコミューン」を含む

のいずれかに該当する行を**自社商品**、それ以外を**メーカー商品**として振り分ける。`商品番号` が空の行はどちらにも含めず「要確認」として別集計する。

## 出力①: 自社データ（`item-data` ルール・XLSX）

自社商品として振り分けられた行のみを対象に、商品データxlsxの品番マスタ・商品分類シートから商品名・大分類/中分類/小分類・楽天商品URLベースの検索コードを付与し、以下12列のXLSX（シート名「販売明細」）として出力する。

`販売日 / 大分類 / 中分類 / 小分類 / 商品名 / 商品番号 / 税抜単価 / 数量 / 小計（税抜）/ 店舗 / 検索コード / マスター一致`

ファイル名は実データの出荷日（無ければ注文日時）の範囲から自動生成（例: `2026年EC販売データ（7月1日〜7月7日）.xlsx`）。[ranking](https://elcommun.github.io/Claude-App/ranking/) アプリへの取込用（`item-data/update_sales.py` で `PRELOADED` に反映）。

## 出力②: メーカー変換データ（`ec-maker-only` ルール・XLSX）

メーカー商品として振り分けられた行のみを対象に、`商品番号` の接頭辞を内蔵の掛け率表（37件、未登録時はブランド名・作家名等のキーワードからフォールバック推定）と照合してメーカー名・掛け率を付与し、`項目・選択肢` 列から「デザイン:」「カラー:」の値だけを抽出、以下10列のXLSX（シート名「メーカー変換」）として出力する。

`店舗 / 出荷日 / 商品名 / 項目・選択肢 / 商品番号 / 税抜単価 / 数量 / 小計（税抜）/ メーカー / 掛け率`

税抜単価・掛け率表・フォールバック推定ルールの詳細は `ec-maker-only/README.md` を参照（本アプリも同じ `MAKER_MAP` / `makerFallback` を使用）。ファイル名は `{年}年EC販売データ（{開始月}月{開始日}日〜{終了月}月{終了日}日）_メーカー変換.xlsx`。[supplier-ranking](https://elcommun.github.io/Claude-App/supplier-ranking/) アプリへの取込用。

## 確認用Excel

変換テーブルのすぐ下に、確認用のリンクを1つ用意（誤判定・未登録の確認用）。クリックすると1つのExcel（`_確認用.xlsx`）に以下の2シートをまとめてダウンロードする（該当行が無いシートは省略）:

- **掛け率未設定**（メーカー変換テーブル形式。接頭辞が掛け率表に未登録のメーカー行）
- **商品番号空**（元のCSV列のまま。要確認・両ファイルとも未集計）

## 税抜単価（両ファイル共通）

`税込別` が「込」の行は `商品単価 ÷ 1.1`（四捨五入）で税抜換算。それ以外は `商品単価` をそのまま使用。`小計（税抜）＝ 税抜単価 × 数量`。

## 更新時の注意

- 商品番号抽出ロジック（`extractProductCode` / `normalizeCode` / `PN_OVERRIDE` / `QOO10_PN_OVERRIDE`）、メーカー掛け率表（`MAKER_MAP` / `makerFallback`）は `item-data` / `ec-maker-only` と重複して持っている。判定ルールやマスタを更新する場合は、3アプリ間の整合性に注意すること（できれば3ファイルとも同じ内容に揃える）。
- `index.html` を更新したらヘッダーの `v○` を1つ増やすこと。
