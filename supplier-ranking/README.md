# 仕入れメーカー別 販売ランキング

`ranking/index.html` の仕様（ログイン・サイドバー・KPI・ランキング表・検索・期間/店舗フィルタ・データ追加アップロード）を踏襲したシングルページアプリ。
対象商品が自社品番ではなく **仕入れメーカー商品**（`ec-maker-only` アプリで変換したメーカー商品）である点のみが異なる。

公開URL: `https://elcommun.github.io/Claude-App/supplier-ranking/`

## サイドバーの分類

自社アプリの「カテゴリ」に相当する分類軸として **メーカー名**を使用する。メーカーごとに掛け率・SKU数・販売数量/金額を集計し、ランキング表で商品別の内訳を表示する。

## 初期データ

`data.js`（`PRELOADED`）に 2026-01-05〜2026-06-30 のサンプルデータを埋め込み済み（`ec-maker-only` アプリの変換CSVをそのまま集計）。

- `PRELOADED.makers`: `{メーカー名: 掛け率}`
- `PRELOADED.products`: `{商品ID: [メーカー, 商品名, 項目・選択肢, 商品番号, 税抜単価]}`
- `PRELOADED.records`: `[出荷日, 店舗, 商品ID, 数量, 小計（税抜）]` の配列

## データ追加（今後の月次更新）

ヘッダーの「⬆ データ更新」から、`ec-maker-only` アプリが出力する**メーカー変換CSV**（店舗/出荷日/商品名/項目・選択肢/商品番号/税抜単価/数量/小計（税抜）/メーカー/掛け率の10列、Shift-JIS）をアップロードすると、既存データにマージされる（`localStorage` キー `ec_supplier_uploads_v1`）。同一メーカー・商品名・項目・選択肢の組み合わせは同一商品として集計に統合される。

新しいメーカーの商品を追加取り込みした場合、既存の `data.js` に恒久的に反映したい時はサンプルデータの再生成（CSVから `data.js` を再ビルド）を検討すること。

## ログイン

`ranking/index.html` と共通の認証（`sessionStorage` キー `ec_auth`）。同一オリジンであれば ranking アプリでログイン済みの場合はこのアプリでも認証済み扱いになる。

## 未実装（`ranking` からの主な省略点）

- 商品画像・欠品日列・カタログPDF出力（自社の手帳・カレンダー専用機能のため対象外）
- 前年比較・検索エイリアス辞書（初回サンプルが半年分のみのため対象外。データが蓄積したら追加を検討）

## 更新時の注意

`index.html` を更新したら `.hd-sub` 内のバージョン番号（`v1`など）を1つ増やすこと（PWAキャッシュ更新の仕組みが `ranking` と同様に組み込まれている）。
