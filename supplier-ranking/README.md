# 仕入れメーカー別 販売ランキング

`ranking/index.html` の仕様（ログイン・サイドバー・KPI・ランキング表・検索・期間/店舗フィルタ・データ追加アップロード）を踏襲したシングルページアプリ。
対象商品が自社品番ではなく **仕入れメーカー商品**（`ec-maker-only` アプリで変換したメーカー商品）である点のみが異なる。

公開URL: `https://elcommun.github.io/Claude-App/supplier-ranking/`

## サイドバーの分類

サイドバー上部の「メーカー別／カテゴリ別」タブで分類軸を切り替えられる。

- **メーカー別**: `メーカー名`ごとに掛け率・SKU数・販売数量/金額を集計
- **カテゴリ別**: 商品名からキーワードマッチングで推定した分類（額縁・フレーム、バッグ・ポーチ・巾着、ファブリック・マット、タオル・ハンカチ・ソックス、文具など。ルールは `index.html` の `CATEGORY_RULES` / `categorizeProduct()`）。カテゴリはメーカー横断のため、行ごとにメーカータグを表示する

いずれの軸でもランキング表は最大300件まで表示し、超過時は「上位300件を表示中（全n件）」と注記する（1メーカー・1カテゴリだけで3,000件を超えることがあるため）。

## 商品名・バリエーションのクリーニング（今後のデータ取り込み全てに必須）

**新しい販売データを取り込む際（アップロード機能経由・`data.js` の手動再生成いずれも）は、必ず以下の整形ルールを適用すること。** 一部の月だけ未整形のまま取り込むと、同じ商品なのに表記違いで別SKU扱いになったり、商品名が再び冗長に戻ってしまう。

- **商品名** (`cleanProductName()`):
  - 末尾の括弧付き品番エコー（例: `(apj-af50_70)`、`(8mw-mm110031-110032)`）を除去
  - 「オーダーメイド」「お見積もり」「受注生産」「製作期間」など、サイズ以降に続く定型の販促・注意書き文言以降を切り捨て
  - 「おしゃれ」「インテリア」「個性的」「リビング」「お部屋」「寝室」など、検索SEO目的で埋め込まれた形容詞的なキーワードを除去
- **バリエーション（項目・選択肢）** (`cleanVarName()`):
  - プレースホルダーの「在庫」単体は空欄化
  - 「在庫 / 実際のバリエーション名」形式は「在庫 / 」部分だけ除去し、実際のバリエーション名を残す

`index.html`（アプリ内アップロード処理）と `data.js` 再生成スクリプトで同一のロジックを共有する。新しい表記パターン（例: 別の販促文言や別のプレースホルダー）が見つかった場合は、両方の実装に同じルールを追加すること。

## 初期データ

`data.js`（`PRELOADED`）に 2021-02-22〜2026-06-30 のデータを埋め込み済み（`ec-maker-only` アプリの変換CSV／出力を集計。メーカー・掛け率が未設定の行は除外）。

- `PRELOADED.makers`: `{メーカー名: 掛け率}`
- `PRELOADED.products`: `{商品ID: [メーカー, 商品名, 項目・選択肢, 商品番号, 税抜単価, カテゴリ]}`
- `PRELOADED.records`: `[出荷日, 店舗, 商品ID, 数量, 小計（税抜）]` の配列

## データ追加（今後の月次更新）

ヘッダーの「⬆ データ更新」から、`ec-maker-only` アプリが出力する**メーカー変換CSV**（店舗/出荷日/商品名/項目・選択肢/商品番号/税抜単価/数量/小計（税抜）/メーカー/掛け率の10列、Shift-JIS）をアップロードすると、既存データにマージされる（`localStorage` キー `ec_supplier_uploads_v1`）。同一メーカー・商品名・項目・選択肢の組み合わせは同一商品として集計に統合される。

新しいメーカーの商品を追加取り込みした場合、既存の `data.js` に恒久的に反映したい時はサンプルデータの再生成（CSVから `data.js` を再ビルド）を検討すること。

## ログイン

`ranking/index.html` と共通の認証（`sessionStorage` キー `ec_auth`）。同一オリジンであれば ranking アプリでログイン済みの場合はこのアプリでも認証済み扱いになる。

## 未実装（`ranking` からの主な省略点）

- 商品画像・欠品日列・カタログPDF出力（自社の手帳・カレンダー専用機能のため対象外）
- 検索エイリアス辞書（要望が出たら追加を検討）

前年比較・ヒートマップはサイドバーのトグルボタンとして実装済み（前年比較は選択期間が前年期間と重ならないよう自動調整するため、1年程度の期間選択で使うのが最も意味がある）。

## 更新時の注意

`index.html` を更新したら `.hd-sub` 内のバージョン番号（`v1`など）を1つ増やすこと（PWAキャッシュ更新の仕組みが `ranking` と同様に組み込まれている）。
