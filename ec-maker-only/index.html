処理ルール

  • A判定：各行の 商品番号 を内部品番に解決し（item-data 準拠）、自社品番マスタ（約0件）に一致、elco- cal-・自社確定接頭辞（X?DR- CAL- EGP- WCL- 等）に一致、または商品名に「エルコミューン」を含む行をプロパー（自社）として削除。それ以外をメーカーとして残します。
  • B税抜単価：税込別 が「込」の行は 商品単価 ÷ 1.1（四捨五入）で税抜換算。それ以外は 商品単価 をそのまま使用。小計（税抜）＝ 税抜単価 × 数量。
  • Cメーカー・掛け率：商品番号 の接頭辞を掛け率表（内蔵36件）と照合して入力。未登録の場合はフォールバックとして、商品名に「プラス」「キングジム」→MDS、「クルテク」→パドルビー、「cocochi」→cocochi、「くるみん」「竹久夢二」→京佑キョウユウ、商品番号が大文字MM/KFで始まる5文字→ユニコン、と推定。それでも未登録の場合は空欄（件数を下部に表示）。
  • D項目・選択肢：元データの 項目・選択肢 列から「デザイン:」「カラー:」等のバリエーション値だけを抽出（送料・注意事項等の行は除外）。
  • E出力：店舗 / 出荷日 / 商品名 / 項目・選択肢 / 商品番号 / 税抜単価 / 数量 / 小計（税抜）/ メーカー / 掛け率 の10列の変換テーブル（Shift-JIS CSV）。
