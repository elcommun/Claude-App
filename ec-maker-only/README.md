# メーカー販売データ変換 — プロパー(自社)除外・税抜/掛け率付与

`item-data` / `ec-inventory` の派生アプリ。`index.html` 一枚のシングルページアプリ。
**受注・売上データ CSV** を取り込み、**プロパー（自社）商品を除外**したうえで、メーカー商品の販売データを
**税抜単価・小計・メーカー名・掛け率を付与した9列の変換テーブル**として出力する。

公開URL: `https://elcommun.github.io/Claude-App/ec-maker-only/`

## 入力データ

| ファイル | 必須 | 役割 |
|---------|------|------|
| **受注・売上データ CSV** | 必須 | `販売店舗 / 出荷日 / 商品名 / 商品番号 / SKU管理番号 / 個数 / 商品単価 / 税込別 …` を含む全店舗の受注/売上一覧。Shift-JIS（CP932）想定。 |
| **商品データ xlsx（自社品番マスタ）** | 任意 | 自社品番の判定に使用。未アップロード時は内蔵スナップショット（`../ec-inventory/genka_master.js`、約9,700件）。最新の自社新商品を反映したい時だけアップロード。 |

## 出力（変換テーブル・9列）

`店舗 / 出荷日 / 商品名 / 商品番号 / 税抜単価 / 数量 / 小計（税抜）/ メーカー / 掛け率`

- **税抜単価**: `税込別` が「込」の行は `商品単価 ÷ 1.1`（四捨五入）で税抜換算。それ以外は `商品単価` をそのまま。
- **小計（税抜）** = `税抜単価 × 数量`。
- **メーカー / 掛け率**: `商品番号` の接頭辞を掛け率表と照合して入力（下表）。
- Shift-JIS（CP932）CSV（`{元ファイル名}_メーカー変換.csv`）。

## 判定ルール（自社の除外・`item-data` 準拠）

各行の `商品番号` を内部品番に解決（`extractProductCode` / `normalizeCode` ＋ Amazon/Qoo10 のハードコードマップ）し、
自社品番マスタ一致・`elco-`/`cal-`・自社確定接頭辞（`X?DR-` `CAL-` `EGP-` `LBG-` `AMP-C` `SFD-` `MK-` `CRB-` `WCL-`、大文字 `AK-`）に該当する行を **プロパー（自社）として除外**。それ以外をメーカー行として変換する。
