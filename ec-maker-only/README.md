# メーカー販売データ変換 — プロパー(自社)除外・税抜/掛け率付与

`item-data` / `ec-inventory` の派生アプリ。`index.html` 一枚のシングルページアプリ。
**受注・売上データ CSV** を取り込み、**プロパー（自社）商品を除外**したうえで、メーカー商品の販売データを
**税抜単価・小計・メーカー名・掛け率・バリエーション情報を付与した10列の変換テーブル**として出力する。

公開URL: `https://elcommun.github.io/Claude-App/ec-maker-only/`

## 入力データ

| ファイル | 必須 | 役割 |
|---------|------|------|
| **受注・売上データ CSV** | 必須 | `販売店舗 / 出荷日 / 商品名 / 商品番号 / SKU管理番号 / 個数 / 商品単価 / 税込別 …` を含む全店舗の受注/売上一覧。Shift-JIS（CP932）想定。 |
| **商品データ xlsx（自社品番マスタ）** | 任意 | 自社品番の判定に使用。未アップロード時は内蔵スナップショット（`../ec-inventory/genka_master.js`、約9,700件）。最新の自社新商品を反映したい時だけアップロード。 |

## 出力（変換テーブル・10列）

`店舗 / 出荷日 / 商品名 / 項目・選択肢 / 商品番号 / 税抜単価 / 数量 / 小計（税抜）/ メーカー / 掛け率`

- **税抜単価**: `税込別` が「込」の行は `商品単価 ÷ 1.1`（四捨五入）で税抜換算。それ以外は `商品単価` をそのまま。
- **小計（税抜）** = `税抜単価 × 数量`。
- **項目・選択肢**: 元データの `項目・選択肢` 列（複数行・`ラベル:値`形式）から、ラベルに「デザイン」または「カラー」を含む行の**値だけ**を抽出（例 `デザイン:kukka（SVT-001）` → `kukka（SVT-001）`、`カラー:シルバー-0020178336` → `シルバー-0020178336`）。送料・注意事項などそれ以外のラベルの行はすべて削除。複数該当する場合は ` / ` で連結。
- **メーカー / 掛け率**: `商品番号` の接頭辞を掛け率表と照合して入力（下表）。未登録の場合は商品名・商品番号のパターンからフォールバック推定（下記）。
- Shift-JIS（CP932）CSV（`{元ファイル名}_メーカー変換.csv`）。
- 変換テーブルのすぐ下に、以下を確認できる目立たない小さいリンクを用意（誤判定・未登録の確認用）:
  - **プロパー(自社)削除分**（元のCSV列のまま、`{元ファイル名}_プロパー削除分.csv`）
  - **掛け率未設定の行**（変換テーブル形式、`{元ファイル名}_掛け率未設定.csv`）
  - **商品番号が空の行**（元のCSV列のまま、`{元ファイル名}_商品番号空.csv`）

## 判定ルール（自社の除外・`item-data` 準拠）

各行の `商品番号` を内部品番に解決（`extractProductCode` / `normalizeCode` ＋ Amazon/Qoo10 のハードコードマップ）し、
自社品番マスタ一致・`elco-`/`cal-`・自社確定接頭辞（`X?DR-` `CAL-` `EGP-` `LBG-` `AMP-C` `SFD-` `MK-` `CRB-` `WCL-`、大文字 `AK-`）に該当する行、または**商品名に「エルコミューン」を含む**行を **プロパー（自社）として除外**。それ以外をメーカー行として変換する。

## メーカー・掛け率のフォールバック推定

`商品番号` の接頭辞が掛け率表に無い場合、以下の順でメーカーを推定する（`index.html` の `makerFallback`）:

| 条件 | メーカー | 掛け率 |
|---|---|---|
| 商品名に「プラス」または「キングジム」を含む | MDS | 0.55 |
| 商品名に「クルテク」を含む | パドルビー | 0.55 |
| 商品名に「cocochi」を含む | cocochi | 0.5 |
| 商品名に「くるみん」または「竹久夢二」を含む | 京佑キョウユウ | 0.6 |
| 商品名に「Alinea」を含む | BEEM | 0.45 |
| 商品番号が大文字 `MM` または `KF` で始まる5文字 | ユニコン | 0.53 |

いずれにも該当しない場合は掛け率表未登録として扱う（メーカー・掛け率は空欄）。
