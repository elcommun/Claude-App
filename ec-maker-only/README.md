# メーカー商品のみ抽出 — プロパー(自社)除外アプリ

`item-data` / `ec-inventory` の **逆**にあたる単機能アプリ。`index.html` 一枚のシングルページアプリ。
**受注・売上データ CSV** を取り込み、**プロパー（自社）商品の行を削除して、メーカー商品の行だけを残した CSV** を出力する。

公開URL: `https://elcommun.github.io/Claude-App/ec-maker-only/`

## 入力データ

| ファイル | 必須 | 役割 |
|---------|------|------|
| **受注・売上データ CSV** | 必須 | `販売店舗 / 商品名 / 商品番号 / SKU管理番号 / 個数 …` を含む全店舗の受注/売上一覧。プロパー抽出アプリ（item-data）と同じ入力。Shift-JIS（CP932）想定。 |
| **商品データ xlsx（自社品番マスタ）** | 任意 | 自社品番の判定に使う。未アップロード時はアプリ内蔵スナップショット（`../ec-inventory/genka_master.js`、約9,700件）を使用。最新の自社新商品を判定へ反映したい時だけアップロード。 |

## 出力

- アップロードした CSV と **同じ列・同じ並び順をそのまま維持**し、メーカー商品の行だけを残した CSV（`{元ファイル名}_メーカーのみ.csv`）。
- 文字コードは入力に合わせて **Shift-JIS（CP932）**。セルの値は無加工でそのまま出力する。

## 判定ルール（`item-data` 準拠）

各行の `商品番号` を内部品番に解決（`item-data` の `extractProductCode` / `normalizeCode` と同一ロジック＋Amazon/Qoo10 のハードコードマップ）したうえで:

- **プロパー（自社・削除）**: 次のいずれか。
  - `商品番号` が `elco-` / `cal-` で始まる
  - 解決した品番（または `SKU管理番号`）が **自社品番マスタ**に存在する
  - 解決した品番が自社確定接頭辞（`X?DR-` `CAL-` `EGP-` `LBG-` `AMP-C` `SFD-` `MK-` `CRB-` `WCL-`、大文字 `AK-`）に一致する
  - ※ 手帳系（`ma-` `mg-` `mh-` `wb-` `wk-` `ws-` 等）は `DR-XX-NNN` に解決されマスター一致 → 自社。
- **メーカー（残す）**: 上記で自社と判定されなかった他社ブランド商品（`8mw-` `wh-` `ldw` `psm-` `ak-`（小文字）`sh-` `tri-` `SLW-`（SLOWER）`MO`（クルテク）等）。
- **取り寄せ**: `apj-` / `rft-` は非在庫・取り寄せ。**既定では削除**。チェックボックスで「残す」に切替可能。
- **商品番号空**: 判定不可として除外。

## メモ

- 価格・原価・在庫の突合は行わない（純粋な行フィルタ）。
- 自社判定ロジックは `item-data/index.html`、自社品番マスタは `ec-inventory/genka_master.js` を流用。これらが更新されれば判定も追随する。
- `index.html` を更新したらヘッダーの `v○` を1つ増やすこと。
