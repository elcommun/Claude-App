# EC棚卸し — 自社/メーカー振り分け出力アプリ

`ranking` などと同じく `index.html` 一枚のシングルページアプリ。
EC在庫データ（itemdata CSV）を取り込み、**自社商品**と**メーカー商品**の2シートに振り分けて、棚卸し用Excelを出力する。

公開URL: `https://elcommun.github.io/Claude-App/ec-inventory/`

**⚠️ 未入荷の予約商品に注意**: 手帳・カレンダーなど、まだ入荷していない予約商品が itemdata に含まれている場合は、棚卸しデータを生成する前に itemdata CSV から該当行を削除すること（アプリでは判別できないため、ページ上部に目立つ注意書きを表示している）。

## 入力データ（毎回アップロード）

| # | ファイル | 必須 | 役割 |
|---|---------|------|------|
| 1 | **itemdata CSV** | 必須 | 在庫数・商品名・システム連携用SKU番号。全商品の一覧。 |
| 2 | **dl-normal-item CSV**（楽天DL） | 必須 | `通常購入販売価格`＝**メーカー商品の上代**。`システム連携用SKU番号` で突合。連携SKUが空の行（選択肢の無い単体商品）は `商品管理番号（商品URL）` で参照（単価が一意の場合のみ）。`psm-brc` は選択肢別在庫でも特例として `商品管理番号（商品URL）` で参照。 |
| 3 | **商品データ xlsx**（原価マスタ） | 任意 | 自社商品の**原価**。未アップロード時はアプリ内蔵スナップショット（`genka_master.js`）を使用。新商品・最新原価を反映したい時のみアップロード。 |

いずれの CSV も Shift-JIS（CP932）想定。

**itemdata を GoQ System からダウンロードする手順**: [GoQ System 在庫連携](https://stock2.goqsystem.com/?nav_id=stockSituation) を開き、「すべて」にチェック → プルダウンで「EL COMMUN 楽天市場店」を選択 → 「CSV出力」ボタンを押す。

**dl-normal-item を RMS でダウンロードする際の必須項目**: 商品番号 / システム連携用SKU番号 / 通常購入販売価格 / 商品名 / バリエーション項目名・選択肢

## 出力

**1つのExcel（`YYYYMM_EC棚卸し.xlsx`）に以下の3シートをまとめて出力**:

- **自社** … `商品名 / 品番 / 原価 / 在庫数 / 合計(=原価×在庫数)`
- **メーカー** … `商品名 / JANコード / 商品管理番号 / 商品番号 / SKU管理番号 / システム連携用SKU番号 / バリエーション / 上代 / 掛け率 / 在庫数 / 合計(=上代×掛け率×在庫数)`
- **掛け率** … 接頭辞別の掛け率一覧

自社・メーカーシートとも末尾に合計行を付与。ファイル名の先頭には実行年月（`YYYYMM_`）が付く（例 `202606_EC棚卸し.xlsx`）。
**合計（原価×在庫数 / 上代×掛け率×在庫数）を算出できなかった商品は自社・メーカーシートに含めない**。該当商品は別途「エラー箇所」として `YYYYMM_EC棚卸し_エラー箇所.xlsx` でダウンロードできる。

## 振り分けルール（`ec-monthly` 準拠）

- システム連携用SKU番号の**接頭辞**で判定。
  - メーカー: `4on- 8mw- ak- alsk- bm- cocochi- cre- ky- ldw ma- mgt- nm- pb- psm- rio- sh- tp- tri- wf wh-` など。
  - 自社: `elco-` `cal-` ほか上記以外。
  - **除外（取り寄せ・非在庫）**: `apj-` `rft-` で始まる品番は棚卸し対象外として出力しない。
  - **個別除外**: `elco-crb111-`（CRB-111）と `WPK-101`（`elco-wpk101`）は棚卸し対象外。
- 商品名は itemdata から抽出し、先頭の装飾（`【…】` `＼…／`）と `｜` 以降、HTMLタグを除去。
- 上代・原価の突合は**大文字小文字を無視**（itemdata `-L` ⇔ dl `-l` 等の表記揺れに対応）。
- 自社品番の単体形は正規品番に変換（末尾の `-` を削除・小文字は大文字化）:
  - `elco-ampcNNN-` → `AMP-CNNN`（例 `elco-ampc037-` → `AMP-C037`）
  - `elco-nxt●●●` → `NXT-●●●`（例 `elco-nxt5246go` → `NXT-5246GO`）
  - `elco-wclNNN` → `WCL-NNN`（例 `elco-wcl023` → `WCL-023`）
  - `elco-wpkNNN` → `WPK-NNN`（例 `elco-wpk101` → `WPK-101`）

## 掛け率（接頭辞別・アプリ内で編集可能）

`合計 = 上代 × 掛け率 × 在庫数`。既定値は `dl-normal` の掛け率表に準拠。
表に無い接頭辞（例 `apj-` `rft-`）は合計が空欄になるため、必要に応じてアプリ上の表に追加する（`localStorage` に保存）。

## エラー箇所の確認

合計（原価×在庫 / 上代×掛け率×在庫）を算出できなかった商品（原価なし・上代なし・掛け率なし）は、
生成結果の下部に件数を表示し、**プレビュー**（先頭200件をその場で表形式表示）と
**ダウンロード**（全件を `EC棚卸し_エラー箇所.xlsx` として出力。区分・エラー種別・商品名・品番・連携SKU・在庫・上代・掛け率を収録）ができる。

「メーカー掛け率（接頭辞別）」のカードは普段は折りたたまれており、ヘッダーをクリックすると開く。

## データ更新メモ

- `genka_master.js` は `商品データ20260623.xlsx` の `商品データ` / `plaisir別注` シート（`品番→原価`、約9,700件）から生成したスナップショット。
- マスタ更新時は、Step3 で最新 xlsx をアップロードするか、`genka_master.js` を再生成する。
- `index.html` を更新したらヘッダーの `v○` を1つ増やすこと。
